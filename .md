MTN MoMo API — Beginner's Guide

What Is It?

The MTN Mobile Money (MoMo) API lets developers integrate mobile payments into apps and websites. It has three core products:

Products - these are collections, disbursments and remittances

Directions - You and Customers

Use cases - Accept payments, Sends payments , International transfers

 To Get Your Keys : You have to sign up at https://momodeveloper.mtn.com
Subscribe to a product (e.g., Collections Sandbox)
Go to your Profile and copy your Subscription Key (Ocp-Apim-Subscription-Key)

Always start in the Sandbox — it's free and moves no real money.

Authentication has three steps : 

Do this once per product to get a Bearer token.

Step 1 — Create an API User
POST /v1_0/apiuser
Host: sandbox.momodeveloper.mtn.com

Headers:
  X-Reference-Id: <generate-a-uuid>
  Ocp-Apim-Subscription-Key: <your-key>
  Content-Type: application/json

Body:
{ "providerCallbackHost": "https://webhook.site/your-id" }

→ 201 Created

Step 2 — Create an API Key
POST /v1_0/apiuser/{userId}/apikey
Headers:
  Ocp-Apim-Subscription-Key: <your-key>

→ { "apiKey": "abc123..." }

Step 3 — Get Access Token
Base64-encode userId:apiKey and send as Basic Auth:
POST /collection/token/
Headers:
  Authorization: Basic <base64(userId:apiKey)>
  Ocp-Apim-Subscription-Key: <your-key>

→ { "access_token": "eyJ...", "expires_in": 3600 }

Tokens expire in 1 hour. Cache and reuse them.

To Request a Payment (Collections) - POST /collection/v1_0/requesttopay
Headers:
  Authorization: Bearer <token>
  X-Reference-Id: <new-uuid>
  X-Target-Environment: sandbox
  Ocp-Apim-Subscription-Key: <your-key>

Body:
{
  "amount": "100",
  "currency": "EUR",
  "externalId": "order-001",
  "payer": { "partyIdType": "MSISDN", "partyId": "46733123453" },
  "payerMessage": "Payment for Order #001",
  "payeeNote": "Thank you!"
}

→ 202 Accepted

To Check payment status: GET /collection/v1_0/requesttopay/{X-Reference-Id}

→ { "status": "SUCCESSFUL" | "PENDING" | "FAILED" }

To Send Money (Disbursements)

POST /disbursement/v1_0/transfer
Headers:
  Authorization: Bearer <disbursement-token>
  X-Reference-Id: <new-uuid>
  X-Target-Environment: sandbox
  Ocp-Apim-Subscription-Key: <disbursement-key>

Body:
{
  "amount": "500",
  "currency": "EUR",
  "externalId": "payout-007",
  "payee": { "partyIdType": "MSISDN", "partyId": "46733123453" },
  "payerMessage": "Salary",
  "payeeNote": "Monthly payment"
}

Each product needs its own Subscription Key and API User.

Quick Node.js Example

jsconst axios = require('axios');
const { v4: uuidv4 } = require('uuid');

const BASE = 'https://sandbox.momodeveloper.mtn.com';
const SUB_KEY = 'YOUR_SUBSCRIPTION_KEY';

async function getToken(userId, apiKey) {
  const creds = Buffer.from(`${userId}:${apiKey}`).toString('base64');
  const res = await axios.post(`${BASE}/collection/token/`, {}, {
    headers: { 'Authorization': `Basic ${creds}`, 'Ocp-Apim-Subscription-Key': SUB_KEY }
  });
  return res.data.access_token;
}

Common Errors

401 means expired token and to fix it you need to request a new token

409 means duplicate X-Reference-Id and to fix it always use a fresh UUID

500 means platform error to fix it you have to retry with backoff

Going Live 
Apply for production access on the portal
Pass MTN's KYB (business verification)
Swap base URL → country-specific production URL
Change X-Target-Environment → e.g., mtnghana, mtnuganda
Use real currency codes (GHS, UGX, NGN, XAF…)
Store API keys in environment variables — never in code
